# Health Predictor App (Diabetes & Stroke Risk)

I built this project as a full-stack health prediction app that combines a Flutter mobile UI with Firebase for user management/data storage, and a Flask API that serves machine learning predictions.

The app collects user-provided health inputs, sends them to a backend API, and returns a prediction for:
- **Diabetes risk**
- **Stroke risk**

> Note: This app is for learning and demonstration purposes and is **not** medical advice.

---

## Features

- Flutter-based mobile app UI (Android/iOS)
- Firebase integration for authentication and data handling
- Flask REST API for model inference
- ML-based prediction endpoints for diabetes and stroke
- API testing using Postman

## How the Prediction Works (at a glance)

1. I collect health-related inputs in the Flutter UI.
2. The app sends the inputs to my Flask API endpoint.
3. The API formats the inputs the same way the ML model expects.
4. The model returns a prediction, and the API sends it back as JSON.
5. I display the result in the app.

---

## API Endpoints (example)

> Update these routes to match your actual Flask implementation.

- `POST /predict/diabetes`
- `POST /predict/stroke`

Each request sends JSON input features and receives a JSON response containing the prediction.
